# Word Lists

We have chosen to use terminology in our documentation in specific ways, which are represented in the following word lists.

In addition to these word lists, we use inclusive language throughout our docs to ensure content is accessible and welcoming to all readers.

## Latin abbreviations

Remember we are writing for an international audience. Use:
- “and others”, “and more”, “and so on” (instead of “etc.”)
- “for example” (instead of “e.g.”)
- “that is” (instead of “i.e.”)

## Latin vs. German verbs

Remember we are writing for an international audience. Use:
- Do (instead of “execute”)
- Get (instead of “obtain”)
- Send (instead of “deliver”)
- Use (instead of “utilize”)

## User interface

Remember that the UI may be rendered differently on different devices. Use:
- field
- section
- select (avoid “click” or “click on”)
- view (avoid “tab”)

## Buzzwords to avoid

- Leverage
- Uplevel
- Innovative
- Productivity
- Immersive
- Orchestrate
- Synergistic
- Actionable
- Monetize
- Utilize
- Usage

## Tautologies to avoid

- Developer API (all APIs are for developers)
