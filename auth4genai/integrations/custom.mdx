The Custom OAuth2 Connection allows you to create integrations with any OAuth2-compliant service that isn't available as a built-in connection type.

## Overview

If a specific identity provider or service isn't available out of the box, Auth0 allows you to create Custom Social Connections. This is particularly useful for integrating with proprietary systems, regional services, or specialized OAuth2 providers.

## Configuration

<Steps>
  <Step title="Prepare OAuth2 Provider">
    Ensure your OAuth2 provider supports:
    - Authorization Code flow
    - Standard OAuth2 endpoints (authorize, token)
    - User info endpoint
    - HTTPS for all endpoints
  </Step>
  <Step title="Create Custom Connection">
    1. In Auth0 Dashboard, go to **Authentication** > **Social**
    2. Click **Create Connection** > **Create Custom**
    3. Configure the OAuth2 endpoints and parameters
    4. Set up user profile mapping
  </Step>
  <Step title="Configure Endpoints">
    Set up the required OAuth2 endpoints:
    - **Authorization URL**: Where users are redirected to authenticate
    - **Token URL**: Where authorization codes are exchanged for tokens
    - **User Info URL**: Where user profile data is retrieved
  </Step>
  <Step title="Map User Attributes">
    Configure how user attributes from the provider map to Auth0 user profile fields.
  </Step>
</Steps>

## Required Configuration

### OAuth2 Endpoints
```json
{
  "authorizationURL": "https://provider.com/oauth/authorize",
  "tokenURL": "https://provider.com/oauth/token",
  "userInfoURL": "https://provider.com/api/user",
  "scope": "read:profile read:email"
}
```

### User Profile Mapping
```javascript
{
  "user_id": "id",
  "name": "full_name",
  "email": "email_address",
  "picture": "avatar_url"
}
```

## Testing Your Connection

Before deploying, test your custom connection:

1. **Authentication Flow**: Verify users can authenticate successfully
2. **User Profile**: Ensure user data is correctly mapped
3. **Token Handling**: Test token refresh and expiration
4. **Error Cases**: Test various error scenarios

## Security Considerations

<Warning>
Always validate the OAuth2 provider's security implementation before creating a custom connection.
</Warning>

1. **HTTPS Only**: Ensure all endpoints use HTTPS
2. **State Parameter**: Verify the provider supports the state parameter for CSRF protection
3. **Token Security**: Understand how the provider handles token security
4. **Rate Limits**: Be aware of any rate limits imposed by the provider

## Example: Custom Social Platform

Here's an example configuration for a hypothetical social platform:

```json
{
  "name": "MyPlatform",
  "strategy": "oauth2",
  "options": {
    "authorizationURL": "https://myplatform.com/oauth/authorize",
    "tokenURL": "https://myplatform.com/oauth/token",
    "userInfoURL": "https://myplatform.com/api/v1/user",
    "clientID": "your-client-id",
    "clientSecret": "your-client-secret",
    "scope": "read:profile read:posts",
    "scripts": {
      "fetchUserProfile": "function(accessToken, context, callback) { /* profile mapping */ }"
    }
  }
}
```
