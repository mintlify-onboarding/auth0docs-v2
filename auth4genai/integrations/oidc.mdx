The OpenID Connect (OIDC) connection allows you to integrate with any identity provider that supports the OpenID Connect standard, providing flexibility for custom integrations.

## Overview

OpenID Connect is a simple identity layer on top of the OAuth 2.0 protocol. It allows your application to verify the identity of users and obtain basic profile information in an interoperable and REST-like manner.

## Configuration

<Steps>
  <Step title="Gather Provider Information">
    Obtain the following from your OIDC provider:
    - **Discovery Document URL** (e.g., `/.well-known/openid_configuration`)
    - **Client ID** and **Client Secret**
    - **Issuer URL**
    - Supported **scopes**
  </Step>
  <Step title="Configure Auth0 Connection">
    1. In Auth0 Dashboard, go to **Authentication** > **Enterprise**
    2. Select **OpenID Connect**
    3. Enter the discovery URL and credentials
    4. Configure scopes and custom parameters
    5. Enable **Token Vault** if available
  </Step>
  <Step title="Test Connection">
    Use the Auth0 Dashboard to test the connection before enabling it for your application.
  </Step>
</Steps>

## Standard Claims

OIDC provides standard claims about the authenticated user:

| Claim | Description |
|-------|-------------|
| `sub` | Subject identifier for the user |
| `name` | Full name of the user |
| `email` | Email address |
| `email_verified` | Whether email has been verified |
| `picture` | Profile picture URL |
| `locale` | User's locale |

## Custom Parameters

You can configure additional parameters for your OIDC connection:

```json
{
  "prompt": "login",
  "max_age": "3600",
  "ui_locales": "en-US"
}
```

## Best Practices

1. **Discovery Document**: Use the discovery document URL when possible for automatic configuration
2. **Scope Management**: Only request scopes that your application needs
3. **Error Handling**: Implement proper error handling for authentication failures
4. **Testing**: Thoroughly test the connection with different user scenarios
