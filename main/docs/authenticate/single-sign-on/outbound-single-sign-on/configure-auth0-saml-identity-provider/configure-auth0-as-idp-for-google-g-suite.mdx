Use the following <Tooltip tip="Security Assertion Markup Language (SAML): Standardized protocol allowing two parties to exchange authentication information without a password." cta="View Glossary" href="/glossary?term=SAML">SAML</Tooltip> configuration for Google Workspace.

1. Go to [Dashboard > Applications > Applications](https://manage.auth0.com/#/applications) and either create a new application or click the name of an application to update.
2. Go to the **Addons** tab and enable the **SAML2 Web App** toggle.
3. On the **Settings** tab, set the **Application Callback URL** to: `https://www.google.com/a/{YOUR-GOOGLE-DOMAIN}/acs`.

   <Frame>![Dashboard Applications Applications Addons Tab SAML2 Web App Settings Tab](/images/cdy7uua7fh8z/6dJgYkcOgMZ73HVTkAWt1x/fe9dbbf306e6c587cb3326c00a3b4e1f/2025-02-27_13-59-00.png)</Frame>
4. Paste the following code into the **Settings** text box and click **Debug**.

   ```json lines
   {
         "audience": "https://www.google.com/a/{yourGoogleDomain}/acs",
         "mappings": {
       "nickname": "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name",
         },
         "createUpnClaim": false,
         "passthroughClaimsWithNoMapping": false,
         "mapUnknownClaimsAsIs": false,
         "mapIdentities": false,
         "signatureAlgorithm": "rsa-sha256",
         "digestAlgorithm": "sha256",
         "nameIdentifierFormat": "urn:oasis:names:tc:SAML:2.0:nameid-format:email",
         "nameIdentifierProbes": [
       "http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress"
         ],
       }
   ```


   

   
5. Scroll to the bottom of the page and click **Enable**.
6. On the **Usage** tab, locate **Identity Provider Metadata**, and click **Download** to download the metadata file. You'll need this when you configure Auth0 as the identity provider (IdP).
