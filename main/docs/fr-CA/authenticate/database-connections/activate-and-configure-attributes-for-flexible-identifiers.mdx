<Warning>

L’activation des Identifiants flexibles sur votre locataire peut entraîner des changements perturbateurs dans votre environnement de production. Pour plus d’informations sur les problèmes potentiels et les limitations, consultez [Identifiants et Attributs flexibles](/docs/fr-CA/authenticate/database-connections/flexible-identifiers-and-attributes).

</Warning>

## Activer la configuration des attributs

1. Naviguez vers [**Auth0 Dashboard > Authentication > Base de données**](https://manage.auth0.com/#/connections/database), puis sélectionnez la Connexion de base de données que vous souhaitez activer dans la liste de **Connexions de base de données**.
2. Allez sur l’onglet **Attributs** et choisissez **Activer** sur l’écran **Nouvelle configuration des attributs**, puis confirmez que vous avez testé votre configuration dans un environnement de développement.
3. Sélectionnez **Procéder**. Les nouvelles options de configuration sont disponibles.

### Choisissez attributs sur une nouvelle connexion

Les attributs sont ajoutés aux connexions en naviguant vers [**Auth0 Dashboard > Authentification > Base de données**](https://manage.auth0.com/#/connections/database).

Pour créer une nouvelle connexion, choisissez **+ Créer une connexion à la base de données** et sélectionnez les Attributs à ajouter. Pour ajouter, configurer ou supprimer des Attributs utilisés par une connexion existante, configurez une connexion existante, choisissez l’onglet **Attributs** et suivez les instructions ci-dessous.

### Configurer ou supprimer des Attributs existants sur une connexion

Tous les Attributs peuvent être modifiés ou supprimés à partir de votre <Tooltip href="/docs/fr-CA/glossary?term=auth0-dashboard" tip="Auth0 Dashboard
Principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire">Auth0 Dashboard</Tooltip>. Les utilisateurs ne peuvent pas se connecter, s’inscrire ou réinitialiser leur mot de passe en utilisant l’Attribut supprimé, et un minimum d’un Attribut doit rester actif sur chaque connexion.

Pour modifier ou supprimer un Attribut, naviguez jusqu’à l’onglet Attributs, puis cliquez sur le bouton de menu **…** à côté d’un Attribut et sélectionnez **Configurer** ou **Supprimer** dans le menu déroulant.

### Rétablir une connexion

Pour rétablir l’état initial d’une connexion, utilisez le point de terminaison [Update a connection](https://auth0.com/docs/api/management/v2/connections/patch-connections-by-id) de <Tooltip href="/docs/fr-CA/glossary?term=management-api" tip="Management API
Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire">Management API</Tooltip> avec un objet `options` qui n’inclut pas le paramètre `attributes`, comme indiqué ci-dessous.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

* Si vous utilisez le paramètre `options`, tout l’objet `options` est écrasé. Pour éviter des données partielles ou d’autres problèmes, assurez-vous que tous les paramètres actuels sont présents en copiant le contenu dans l’objet `options` de connexions et en l’incluant dans l’appel PATCH. Le seul paramètre que vous devez supprimer est `attributes`.
* Si `username` est activé en tant qu’attribut pour votre locataire, l’exécution de l’action suivante nécessite que vous modifiiez vos paramètres de `username`. Vous devrez reconfigurer vos paramètres de nom d’utilisateur si vous choisissez cette action.

</Callout>

{
"options": {
       }
}
