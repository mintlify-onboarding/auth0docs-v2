La Connexion intégrée permet à vos aux utilisateurs de se connecter directement à votre application, tout en transmettant leurs identifiants au serveur Auth0 pour authentification. Nous ne recommandons pas d’utiliser la Connexion intégrée. Pour en savoir plus, consultez [Connexion universelle centralisée par rapport à la Connexion intégrée](/docs/fr-CA/authenticate/login/universal-vs-embedded-login).

Si vous choisissez d’utiliser la Connexion Intégrée, [vous devez configurer votre application pour le Cross-Origin Resource Sharing (CORS)](/docs/fr-CA/get-started/applications/set-up-cors) et [mettre en place un domaine personnalisé](/docs/fr-CA/customize/custom-domains). Vous pouvez ensuite mettre en œuvre le [Trousse SDK pour Lock](/docs/fr-CA/libraries/lock) ou [Trousse SDK pour Auth0.js](/docs/fr-CA/libraries/auth0js) dans votre application, ou appeler la [Auth0 Authentication API](https://auth0.com/docs/api/authentication) directement.

Auth0 prend en charge la Connexion intégrée avec des connexions sans mot de passe pour plusieurs types d’applications :

* [Connexion sans mot de passe intégrée dans les applications natives](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/native)
* [Connexion sans mot de passe intégrée dans les applications Web classiques](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/webapps)
* [Authentification sans mot de passe intégrée pour les applications à page unique](/docs/fr-CA/authenticate/passwordless/implement-login/embedded-login/spa)
