Auth0 vous permet de configurer votre propre fournisseur de messagerie SMTP afin que vous puissiez gérer, surveiller et dépanner plus en détail vos communications par courriel. L’infrastructure de messagerie intégrée d’Auth0 ne doit être utilisée que pour les courriels de test.

## Prérequis

* Pour que les courriels puissent être envoyés de Auth0 à votre SMTP, des ports doivent être ouverts et il faut autoriser les connexions entrantes provenant d’adresses IP spécifiques. Pour mettre à jour la liste des adresses IP, accédez à [Dashboard > Image de marque > Fournisseur de messagerie électronique](https://manage.auth0.com/#/branding/email_provider). Voir [Ajouter des adresses IP à AllowList](/docs/fr-CA/secure/security-guidance/data-security/allowlist) pour plus de détails.
* Vous devez disposer d’un compte [Amazon AWS](http://aws.amazon.com/ses/).
* Déterminez les identifiants que vous souhaitez utiliser : API ou SMTP. Pour plus d'informations sur les identifiants SES, consultez la page [Utilisation d'identifiants avec Amazon SES](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/using-credentials.html). d'Amazon.

## Configurer Amazon SES avec les informations d’identification de l’API

1. Connectez-vous à votre compte [Amazon AWS](http://aws.amazon.com/ses/).
2. [Vérifiez votre domaine](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/verify-domains.html).
3. [Vérifier les adresses courriel](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/verify-email-addresses.html)
4. [Demandez l’accès à la production](https://docs.aws.amazon.com/ses/latest/DeveloperGuide/request-production-access.html?icmpid=docs_ses_console).
5. [Obtenez vos clés d’accès AWS.](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/get-aws-keys.html). Copiez ces clés; vous devrez saisir ces valeurs dans Auth0.
6. [Joindre une politique avec les autorisations appropriées](http://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_manage.html). Joindre une politique avec les autorisations `ses:SendRawEmail` et `ses:SendEmail`, comme dans cet exemple :

   ```json lines
   {
       "Version": "2012-10-17",
       "Statement": [
           {
               "Effect": "Allow",
               "Action": [
                   "ses:SendRawEmail",
                   "ses:SendEmail"
               ],
               "Resource": "*"
           }
       ]
   }
   ```

   

   
7. Naviguez vers Auth0 [Dashboard > Image de marque > Fournisseur de messagerie électronique](https://manage.auth0.com/#/branding/email_provider).
8. Activez la bascule **Utiliser mon propre fournisseur de messagerie**.
9. Cliquez sur le logo **Amazon Web Services**.
10. Fournissez une adresse courriel de provenance **De** , saisissez votre **Identifiant de la clé d’accès** et **Clé d’accès secrète** AWS et sélectionnez votre **Région**.
11. Cliquez sur **Enregistrer**.

### Configurer Amazon SES avec les informations d’identification de l’SMTP

1. Inscrivez-vous pour un compte [Amazon AWS](http://aws.amazon.com/ses/) ou connectez-vous.
2. [Vérifiez votre domaine](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/verify-domains.html).
3. [Demandez l’accès à la production](https://docs.aws.amazon.com/ses/latest/DeveloperGuide/request-production-access.html?icmpid=docs_ses_console).
4. [Obtenir vos informations d’identification SMTP.](http://docs.aws.amazon.com/ses/latest/DeveloperGuide/smtp-credentials.html). Copiez les informations d’identification de sécurité; vous devrez saisir ces valeurs dans Auth0.
5. Accédez à [Dashboard > Courriels > Fournisseur de messagerie personnalisée](https://manage.auth0.com/#/emails/provider).
6. Activez la bascule **Utiliser mon propre fournisseur de messagerie**.
7. Cliquez sur le logo **SMTP**.
8. Fournissez une adresse courriel de provenance **De**, puis saisissez le **Hôte**, **Port**, **Nom d’utilisateur**, et **Mot de passe** de votre serveur SMTP, et cliquez sur **Enregistrer**. Vous pouvez utiliser `email-smtp.us-east-1.amazonaws.com` (en utilisant la région appropriée plutôt que `us-east-1`) pour **Hôte** et `587` comme **Port**.

## Courriel de test

Vous pouvez maintenant envoyer un courriel de test en utilisant le bouton **Envoyer un courriel de test**. Si vous avez tout configuré correctement, vous recevrez un courriel de confirmation. Si vous ne recevez pas de courriel après quelques minutes, veuillez vérifier vos [Journaux Auth0](https://manage.auth0.com/#/logs) pour toute erreur.

La [console Amazon SES](https://console.aws.amazon.com/ses) affichera maintenant les informations de livraison pour tous les courriels envoyés à vos utilisateurs.
