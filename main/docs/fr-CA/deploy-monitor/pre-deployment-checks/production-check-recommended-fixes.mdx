Les contrôles suivants permettent de vérifier si vous avez suivi toutes les recommandations (qui sont facultatives) en vue d’un déploiement réussi en production.

Pour plus d’informations sur les résultats des contrôles, voir [Exécuter les contrôles de production](/docs/fr-CA/deploy-monitor/pre-deployment-checks/how-to-run-production-checks).

<table class="table"><thead>
<tr>
<th>Vérifier</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>Configurer l’exportation des journaux</td>
<td>Assurez-vous que vous transmettez les entrées de journal des locataires à un emplacement de stockage ou à un fournisseur externe. Nous vous recommandons de <a href="https://auth0.com/docs/customize/log-streams">configurer un service d’exportation de journaux</a> pour conserver les journaux pendant plus de 30 jours et permettre une meilleure analyse et alertes.</td>
</tr>
<tr>
<td><a href="/docs/fr-CA/custom-domains">Domaine personnalisé</a> doit être configuré</td>
<td>Utiliser les <a href="https://manage.auth0.com/#/tenant/custom_domains">domaines personnalisés avec la connexion universelle</a> pour une expérience plus transparente et plus sûre pour vos utilisateurs finaux.</td>
</tr>
<tr>
<td><a href="/docs/fr-CA/universal-login/custom-error-pages">Page d’erreur personnalisée</a> doit être configurée</td>
<td><a href="https://manage.auth0.com/#/account">Configurer une page d’erreur personnalisée</a> avec les détails de votre application et la marque de votre entreprise.</td>
</tr>
<tr>
<td>Activer la connexion universelle</td>
<td>Assurez-vous que vous utilisez la dernière version de la <a href="https://auth0.com/docs/authenticate/login/auth0-universal-login/new-universal-login-vs-classic-universal-login">connexion universelle</a>. Nous vous recommandons de mettre à jour votre page de connexion en passant de l’expérience classique à la connexion universelle pour bénéficier d’une meilleure accessibilité et performance, d’une personnalisation sans code et de nouvelles fonctionnalités.</td>
</tr>
<tr>
<td><a href="/docs/fr-CA/email/custom">Modèles de courriel</a> doivent être configurés</td>
<td><a href="https://manage.auth0.com/#/emails">Configurer des modèles de courriels personnalisés</a> avec les détails de votre application et la marque de votre entreprise.</td>
</tr>
<tr>
<td><a href="/docs/fr-CA/mfa">Authentification multifacteur (MFA)</a> doit être configurée</td>
<td>Il est recommandé de considérer l’authentification multifacteur (MFA) comme une pratique sûre pour authentifier vos utilisateurs.</td>
</tr>
<tr>
<td><a href="/docs/fr-CA/tutorials/manage-dashboard-admins">MFA pour les administrateurs des locataires</a> est activée</td>
<td><a href="https://manage.auth0.com/#/account/admins">Activer l’authentification multifacteur (MFA)</a> pour les administrateurs de locataires.</td>
</tr>
<tr>
<td><a href="/docs/fr-CA/logout#set-the-allowed-logout-urls-at-the-account-level">L’URL de redirection de la déconnexion</a> doit être défini</td>
<td>Examinez les <a href="https://manage.auth0.com/#/account/advanced">URL de redirection de la déconnexion autorisées</a> pour votre application.</td>
</tr>
<tr>
<td>Les applications doivent utiliser RS256 au lieu de HS256 pour l’algorithme de signature du jeton Web JSON (JWT)</td>
<td>Définissez [Algorithme de signature] du JWT (/apis#signing-algorithms) sur RS256 au lieu de HS256 pour chaque application.</td>
</tr>
<tr>
<td>Définissez l’<a href="https://auth0.com/docs/get-started/applications/application-settings#application-uris">URI de connexion à l’application</a></td>
<td>Indiquer un <a href="https://auth0.com/docs/get-started/applications/application-settings#application-uris">URI de connexion à l’application</a> pour toutes les applications interactives.</td>
</tr>
<tr>
<td>Définissez l’<a href="https://auth0.com/docs/get-started/applications/application-settings#application-uris">URL de déconnexion autorisées du locataire</a></td>
<td>Créez au moins une route de déconnexion accessible au public et <a href="https://auth0.com/docs/get-started/applications/application-settings#application-uris">mettez à jour les paramètres du locataire</a> pour pointer là.</td>
</tr>
<tr>
<td>Définissez l’<a href="https://auth0.com/docs/get-started/tenant-settings#login-and-logout">URI de connexion du locataire</a></td>
<td>Publiez une route de connexion par défaut et <a href="https://auth0.com/docs/get-started/tenant-settings#login-and-logout">mettez à jour vos paramètres du locataire</a> pour pointer vers cette route.</td>
</tr>
</tbody>
</table>
