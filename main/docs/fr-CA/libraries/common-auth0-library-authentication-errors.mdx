Les actions de vos utilisateurs, ou des données entrées au cours de l’enregisrement ou la connexion, peuvent entraîner des erreurs. Voici une liste des erreurs les plus courantes si vous utilisez une des librairies Auth0 pour l’authentification.

## Inscription

Dans le cas d’un enregistrement non réussi, voici les erreurs les plus courantes :

<table class="table"><thead>
<tr>
<th><strong>Erreur</strong></th>
<th><strong>Description</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>invalid_password</strong></td>
<td>Si le mot de passe utilisé ne respecte pas la politique relative au mot de passe pour la connexion</td>
</tr>
<tr>
<td><strong>invalid_signup</strong></td>
<td>L’utilisateur que vous essayez d’inscrire n’est pas valide.</td>
</tr>
<tr>
<td><strong>password_dictionary_error</strong></td>
<td>Le mot de passe choisi est trop simple.</td>
</tr>
<tr>
<td><strong>password_no_user_info_error</strong></td>
<td>Le mot de passe choisi est basé sur des informations de l’utilisateur.</td>
</tr>
<tr>
<td><strong>password_strength_error</strong></td>
<td>Le <a href="/docs/fr-CA/connections/database/password-strength">mot de passe est trop faible</a></td>
</tr>
<tr>
<td><strong>unauthorized</strong></td>
<td>Si vous n’êtes pas autorisé à vous inscrire à cette application. Peut être lié à la violation d’une règle spécifique.</td>
</tr>
<tr>
<td><strong>user_exists</strong></td>
<td>L’utilisateur que vous essayez d’inscrire est déjà inscrit.</td>
</tr>
<tr>
<td><strong>username_exists</strong></td>
<td>Le nom d’utilisateur avec lequel vous essayez de vous inscrire est déjà utilisé.</td>
</tr>
</tbody>
</table>

## Connexion

Dans le cas d’une connexion non réussie, les erreurs les plus courantes sont :

<table class="table"><thead>
<tr>
<th><strong>Erreur</strong></th>
<th><strong>Description</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>access_denied</strong></td>
<td>Lors de l’utilisation de l’authentification Web, le serveur de ressources refuse l’accès selon les spécifications OAuth2</td>
</tr>
<tr>
<td><strong>invalid_user_password</strong></td>
<td>Le nom d’utilisateur et/ou le mot de passe utilisé pour l’authentification sont invalides</td>
</tr>
<tr>
<td><strong>mfa_invalid_code</strong></td>
<td>Le code <dfn id="react-containers-DefinitionTooltip-0"><span class="tooltip-portal-underlined-word" data-react-universal-portal="">multi-factor authentication (MFA)</span></dfn> fourni par l’utilisateur n’est pas valide ou a expiré</td>
</tr>
<tr>
<td><strong>mfa_registration_required</strong></td>
<td>L’administrateur a demandé une <a href="/docs/fr-CA/mfa">authentification multifacteur (MFA)</a>, mais l’utilisateur ne s’est pas inscrit</td>
</tr>
<tr>
<td><strong>mfa_required</strong></td>
<td>L’utilisateur doit fournir le code d’<a href="/docs/fr-CA/mfa">authentification multifacteur (MFA)</a> pour s’authentifier</td>
</tr>
<tr>
<td><strong>password_leaked</strong></td>
<td>Si le mot de passe a été divulgué et qu’il faut en utiliser un autre</td>
</tr>
<tr>
<td><strong>PasswordHistoryError</strong></td>
<td>Le mot de passe fourni pour l’inscription/la mise à jour a déjà été utilisé (signalé lorsque l’<a href="/docs/fr-CA/connections/database/password-options#password-history">historique des mots de passe</a> est activé)</td>
</tr>
<tr>
<td><strong>PasswordStrengthError</strong></td>
<td>Le mot de passe fourni ne correspond pas aux [exigences de force] de la connexion (/connections/database/password-strength)</td>
</tr>
<tr>
<td><strong>too_many_attempts</strong></td>
<td>Le compte est bloqué à cause du trop grand nombre de tentatives de connexion</td>
</tr>
<tr>
<td><strong>unauthorized</strong></td>
<td>L’utilisateur avec lequel vous essayez de vous connecter est bloqué</td>
</tr>
</tbody>
</table>
