Lorsque vous utilisez votre propre code HTML pour les pages <Tooltip href="/docs/fr-CA/glossary?term=multifactor-authentication" tip="Authentification multifacteur (MFA)
Processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, tel qu’un code par SMS." cta="Voir le glossaire">MFA</Tooltip> avec la connexion classique, le gadget logiciel MFA d’Auth0 présente les limitations suivantes :

* La MFA par courriel n’est pas prise en charge.
* Si les utilisateurs ont enregistré plusieurs facteurs, ils ne peuvent pas sélectionner celui à utiliser; le gadget logiciel MFA les invite à se connecter avec le facteur le plus sécurisé.
* Les fonctionnalités d’[internationalisation](/docs/fr-CA/customize/internationalization-and-localization/universal-login-internationalization) de la connexion universelle ne sont pas utilisées.

Les options de thème pour le gadget logiciel MFA sont répertoriées sous la propriété `theme`.

<table class="table"><thead>
<tr>
<th>Option</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>`icon`</td>
<td>L’URL de l’image de l’en-tête. Par défaut, il s’agit du logo Auth0. La hauteur recommandée est de 58 pixels.</td>
</tr>
<tr>
<td>`primaryColor`</td>
<td>Définit la couleur primaire du gadget logiciel de la MFA. Utile lors de l’utilisation d’un `icon` personnalisé pour s’assurer que des couleurs complémentaires apparaissent. La valeur par défaut est `#ea5323`.</td>
</tr>
</tbody>
</table>

## Exemples

```js lines
return new Auth0MFAWidget({

...

  theme: {
    icon: 'https://example.com/assets/logo.png',
    primaryColor: 'blue'
  },
  
...  
  
})
```

Voir un [exemple de gadget logiciel de base Guardian](https://github.com/auth0/auth0-guardian.js/blob/master/example/basic_widget.html) utilisant [auth0-guardian.js](https://github.com/auth0/auth0-guardian.js). Consultez le fichier [readme](https://github.com/auth0/auth0-guardian.js/tree/master/example) pour plus de détails.
