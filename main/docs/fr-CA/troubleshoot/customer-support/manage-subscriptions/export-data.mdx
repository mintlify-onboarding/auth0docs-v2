Toutes les données de votre locataire Auth0 sont toujours sous votre contrôle et sont disponibles via <Tooltip href="/docs/fr-CA/glossary?term=management-api" tip="Management API
Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire">Management API</Tooltip> à tout moment.

## Mots de passe utilisateur

Les seules informations qui ne sont pas disponibles via l’API (pour des raisons de sécurité) sont les hachages de mots de passe des utilisateurs de votre base de données hébergée par Auth0 et les clés privées. Vous pouvez toujours demander ces informations en ouvrant un [ticket de soutien](https://support.auth0.com). Cette opération n’est pas disponible pour notre niveau d’abonnement gratuit, et nous ne sommes pas en mesure d’accepter ou de garantir les demandes d’exportation à une heure et une date spécifiques.

Si vous stockez les mots de passe des utilisateurs dans votre base de données, configurez une connexion personnalisée à la base de données qu’Auth0 interrogera chaque fois qu’un utilisateur se connectera. Dans ce cas, Auth0 ne stockera jamais de hachage de mot de passe, à moins que vous ne choisissiez de migrer progressivement les utilisateurs vers Auth0.

## Composants Auth0

Auth0 utilise principalement <Tooltip href="/docs/fr-CA/glossary?term=openid" tip="OpenID
Norme ouverte d’authentification qui permet aux applications de vérifier l’identité des utilisateurs sans avoir à collecter leurs informations de connexion." cta="Voir le glossaire">OpenID</Tooltip> Connect (OIDC) comme protocole d’authentification, vous devriez donc être en mesure d’implémenter une intégration à votre application en utilisant des bibliothèques standard. La même situation s’applique lors de l’intégration d’Auth0 via [SAML](/docs/fr-CA/authenticate/single-sign-on/outbound-single-sign-on/configure-auth0-saml-identity-provider). Toutes les trousses SDK, bibliothèques et échantillons d’Auth0  [sont publiés sur GitHub en tant que logiciel libre](https://github.com/auth0/).

## Fournisseurs d’identité sociale

Si vous choisissez de ne pas utiliser Auth0, mais que vous souhaitez continuer à utiliser les mêmes identifiants et secrets clients <Tooltip href="/docs/fr-CA/glossary?term=oath2" tip="OAuth 2.0
Cadre d’applications d’autorisation qui définit les protocoles d’autorisation et les flux de production." cta="Voir le glossaire">OAuth</Tooltip> pour vos fournisseurs d’identité sociale, vous conserverez l’accès aux informations de l’utilisateur sans avoir besoin d’afficher de nouvelles boîtes de dialogue de consentement.

## Code personnalisé

Toutes les fonctionnalités du code personnalisé d’Auth0 (règles, scripts de base de données personnalisés, connexions OAuth personnalisées, etc.) s’exécutent sur un service « sandbox » Node.js. Toutes les bibliothèques disponibles sur le service « sandbox » sont également disponibles sur npm pour une utilisation avec le code Node.js standard.
