Un fichier [HAR](https://en.wikipedia.org/wiki/.har) (HTTP Archive) est un journal au format JSON des interactions d’un navigateur Web avec un serveur Web.

Un fichier HAR montre la séquence de redirections qui se produit lors d’une transaction de connexion, ce qui en fait une excellente ressource pour dépanner les problèmes d’authentification. L’inclusion d’un fichier HAR dans vos [demandes d’soutien](https://support.auth0.com) peut contribuer à accélérer le processus de dépannage.

<Warning>

Avant de partager un fichier HAR avec quiconque (y compris Auth0), assurez-vous de supprimer ou d’obscurcir toutes les données sensibles, telles que :

* les informations confidentielles de l’utilisateur
* les informations personnelles identifiables (PII)
* les informations confidentielles relatives à l’application

Pour en savoir plus, consultez les articles suivants sur [Communauté Auth0](https://community.auth0.com) :

* [Assainissement des traces HTTP](https://community.auth0.com/t/sanitizing-http-traces/119488)
* [Comment assainir automatiquement un fichier de trace HTTP](https://community.auth0.com/t/how-to-sanitize-a-http-trace-file-automatically/120583)
* [Comment caviarder manuellement des informations sensibles](https://community.auth0.com/t/how-to-manually-redact-sensitive-information/122554)
* [Fichier HAR trop volumineux pour être téléversé dans le dossier d’assistance](https://community.auth0.com/t/har-file-is-too-large-to-upload-to-the-support-case/122488)

</Warning>

## Générer des fichiers HAR

Vous pouvez générer un fichier HAR dans la plupart des navigateurs Web à l’aide des outils de développement ou d’inspecteur intégrés.

### Google Chrome

1. Fermez toutes les fenêtres **incognito** dans Google Chrome.
2. Ouvrez une **nouvelle fenêtre incognito** dans Google Chrome.
3. Allez à **Afficher > Développeur > Outils du développeur**.
4. Dans le panneau Outils du développeur, choisissez l’onglet **Réseau**.
5. Vérifiez la case à cocher **Conserver le journal** pour enregistrer toutes les interactions.
6. Visitez la page et suivez les étapes qui déclenchent le problème.
7. Choisissez l’onglet **Réseau**.
8. Cliquez sur la flèche vers le bas pour exporter le fichier HAR.
9. Enregistrez le fichier HAR.

### Safari

1. Assurez-vous que la case à cocher **Afficher le menu Développer dans la barre de menus** est cochée sous **Safari > Préférences > Avancé**.
2. Choisissez **Fichier> Ouvrir une nouvelle fenêtre privée**.
3. Choisissez **Show Web Inspector (Afficher l’inspecteur Web)** à partir du menu **Developer (Développeur)**, puis naviguez jusqu’à l’onglet **Network (Réseau)**.
4. À droite de **All (Tous)** en haut à gauche de l’inspecteur Web, cliquez sur le bouton **Filter (Filtrer)** et choisissez **Preserve Log (Conserver le journal).**
5. Visitez la page Web sur laquelle le problème se produit.
6. Choisissez **Develop (Développeur) > Show Web Inspector (Afficher l’inspecteur Web)**. La fenêtre Web Inspector (Inspecteur Web) s’affiche.
7. Suivez les étapes de la page qui déclenchent le problème.
8. Sélectionnez l’onglet **Réseau**.
9. Cliquez sur **Exporter** situé sur le côté supérieur droit du panneau.
10. Enregistrez le fichier HAR.

### Firefox

1. Fermez toutes fenêtres **privées** dans Firefox.
2. Ouvrez une **nouvelle fenêtre privée** dans Firefox.
3. Allez à **Outils > Développeur > Réseau** (**Ctrl + Shift + E**).
4. Cliquez sur l’onglet **Réseau** et choisissez **Conserver les journaux.**
5. Visitez la page et suivez les étapes qui déclenchent le problème.
6. Choisissez l’onglet **Réseau** et faites un clic droit, puis sélectionnez **Enregistrer tout en tant que Har**.
7. Enregistrez le fichier HAR.

### Microsoft Edge

1. Fermez toutes les fenêtres **InPrivate** dans Microsoft Edge.
2. Ouvrez une **nouvelle fenêtre InPrivate** (**Ctrl + Shift + N**).
3. Allez à **Paramètres et plus** (l’icône **...**), puis choisissez **Plus d’outils > Outils du développeur**.
4. Démarrez une session de profilage sous l’onglet **Network** (Réseau) de la barre d’outils Developer Tools (Outils du développeur).
5. Visitez la page et suivez les étapes qui déclenchent le problème.
6. Sélectionnez Exporter comme HAR (**Ctrl + S**) et enregistrez le fichier HAR.

### Internet Explorer

1. Fermez toutes les fenêtres **InPrivate** dans Internet Explorer.
2. Ouvrez une **nouvelle fenêtre InPrivate** dans Internet Explorer (**Ctrl + Shift + P**).
3. Allez à **Outils > F12 / Options développeur / F12 > Réseau**.
4. Assurez-vous que l’option **Effacer les entrées lors de la navigation** est désactivée.
5. Visitez la page et suivez les étapes qui déclenchent le problème.
6. Choisissez l’onglet **Réseau** et sélectionnez **Exporter comme HAR** (**Ctrl + S**).
7. Enregistrez le fichier HAR.

## Analysez les fichiers HAR

Auth0 ne fait pas usage et n’approuve pas l’utilisation d’applications, d’outils ou de logiciels tiers pour analyser les fichiers HAR.

Après avoir nettoyé un fichier HAR, vous pouvez l’ouvrir dans un éditeur de texte ou un navigateur pour afficher les événements enregistrés.

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

Consultez la [page Surveillance](/docs/fr-CA/deploy-monitor/monitor) pour plus de détails sur chacun des événements du journal qui peuvent vous aider à résoudre les problèmes.

</Callout>
