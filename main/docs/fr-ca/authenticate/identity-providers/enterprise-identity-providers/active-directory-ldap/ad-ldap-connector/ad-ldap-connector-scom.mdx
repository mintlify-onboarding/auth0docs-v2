Le connecteur Auth0 AD/LDAP peut s’exécuter en tant que service sur des machines Windows.

Vous pouvez surveiller l’état du service à l’aide de System Center comme vous le feriez avec n’importe quel autre service.

1. Ouvrir le **Assistant d’ajout de surveillance** et sélectionnez **Type de surveillance : Service Windows** :

   <Frame>![Monitor AD/LDAP Connector Wizard Screen 1](/docs/images/fr-ca/cdy7uua7fh8z/6234mP3Hx1g3Iwf7y9mKH9/7a2ac217f7d22070c20a769939461f75/ss-2014-12-11T22-48-51.png)</Frame>

2. Saisissez un nom et une description :

   <Frame>![Monitor AD/LDAP Connector Wizard Screen 2](/docs/images/fr-ca/cdy7uua7fh8z/2KZkQoBuubAgAEWBPnkJx0/abdcdf5e1e0fad20829b255401db523f/ss-2014-12-11T22-49-57.png)</Frame>

3. Sélectionnez le serveur sur lequel le connecteur AD/LDAP est installé, puis sélectionnez **Auth0 ADLDAP**:

   <Frame>![Monitor AD/LDAP Connector Wizard Screen 3](/docs/images/fr-ca/cdy7uua7fh8z/DOQHiJYYiJKZrWzHYUIiB/21eae1ad8435a0754c6e221d5e33bab5/ss-2014-12-11T22-50-37.png)</Frame>

4. Sélectionnez les limites de **Processeur** et **Limites de mémoire**. Par exemple, 10 % de CPU et 200 Mo de RAM sont de bonnes limites pour déclencher des alertes.

Nous vous recommandons également de configurer une transaction synthétique pour surveiller l’authentification de bout en bout.

Pour en savoir plus, consultez [Surveiller Auth0 à l’aide de SCOM](/docs/fr-ca/deploy-monitor/monitor/monitor-using-scom).
