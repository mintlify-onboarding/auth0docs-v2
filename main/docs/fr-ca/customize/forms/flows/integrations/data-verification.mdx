Cette liste d’activités de vérification des données vous permet de vérifier les adresses courriel et de générer et vérifier des mots de passe à usage unique.

## Génération d’un mot de passe à usage unique

Génère un code numérique d’une validité de cinq minutes qui peut être envoyé par courriel, SMS ou toute autre application de messagerie.

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/1h69FWzZwk9beKKjB4Cq38/16aa515e61cc6e9ebdfd0849201e3141/generate-one-time-password.png)</Frame>

### Paramètres d’entrée

<table class="table">
<thead>
<tr>
<th><b>Paramètre</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>Valeur de référence (obligatoire)</td>
<td> 
Vous pouvez lier votre code généré à un numéro de téléphone, une adresse courriel ou n’importe quelle valeur.
          </td>
</tr>
<tr>
<td>Longueur du code (obligatoire)</td>
<td>
Nombre de chiffres du code généré.
          </td>
</tr>
</tbody>
</table>

### Objet de sortie

<table class="table">
<thead>
<tr>
<th><b>Propriété</b></th>

</tr>

<td>Chaîne</td>
<td>Code généré</td>
</tr>
</tbody>
</table>

### Exemple d’objet de sortie

```javascript lines
{
  "code": "███",
}
```

## Vérification du mot de passe à usage unique

Vérifie si le code du mot de passe à usage unique fourni est valide ou non.

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/3bkT8QDN4dcl0YHTrSOIGj/9698a07f55353a11ca98164377cd4217/verify-otp.png)</Frame>

### Paramètres d’entrée

<table class="table">
<thead>
<tr>
<th><b>Paramètre</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>Valeur de référence (obligatoire)</td>
<td> 
Utiliser la même valeur de référence configurée dans Générer un mot de passe à usage unique.
          </td>
</tr>
<tr>
<td>Longueur du code (obligatoire)</td>
<td>
Nombre de chiffres du code généré.
          </td>
</tr>
</tbody>
</table>

### Objet de sortie

<table class="table">
<thead>
<tr>
<th><b>Propriété</b></th>
<th><b>Type</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>`valid`</td>
<td>Booléen</td>
<td>Indique si le code fourni est valide ou non en renvoyant une valeur `true` ou `false`.</td>
</tr>
<tr>
<td>`result`</td>
<td>Chaîne</td>
<td>
Renvoie différents codes de résultat en fonction du code fourni :
            <ul> 
            <li> `MISSING:` Le code fourni n’existe pas.</li>
<li> `EXPIRED:` Le code fourni a expiré.</li>
<li> `VALID:` Le code fourni est valide.</li>
<li> `UNEQUAL:` Le code fourni n’est pas valide.</li>
</ul></td>
</tr>
</tbody>
</table>

### Exemples d’objets de sortie

```javascript lines
{
  "valid": false,
  "result": "UNEQUAL"
}
```

```javascript lines
{
  "valid": true
}
```

## Vérification de l’adresse courriel

Effectue un certain nombre de règles de vérification sélectionnées par rapport à l’adresse courriel fournie.

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/6qkqny1hhObFalHwdEG342/1a2e18195b29102c3f68744f42277917/verify-email.png)</Frame>

### Paramètres d’entrée

<table class="table">
<thead>
<tr>
<th><b>Paramètre</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>Courriel (obligatoire)</td>
<td>
adresse courriel à vérifier.
          </td>
</tr>
<tr>
<td>Enregistrement MX requis</td>
<td>
Enregistrements MX requis Lorsqu’un domaine ne dispose pas d’enregistrements MX, les serveurs de messagerie ne savent pas où envoyer les courriels pour ce domaine.
          </td>
</tr>
<tr>
<td>Bloquer les fournisseurs d’adresses courriel gratuits</td>
<td>
Bloquer les adresses courriel gratuites, telles que @gmail.com, @hotmail.com, etc.
          </td>
</tr>
<tr>
<td>Bloquer les fournisseurs d’adresses courriel temporaires</td>
<td>Bloquer les adresses courriels temporaires, telles que @mailnator.com, @maildrop.cc, etc.</td>
</tr>
<tr>
<td>Bloquer les alias d’adresses courriel</td>
<td>Bloquer les adresses comportant des alias, par exemple jane.doe+alias@gmail.com</td>
</tr>
<tr>
<td>Liste des domaines autorisés</td>
<td>Autoriser uniquement les courriels provenant des domaines approuvés dans votre liste.</td>
</tr>
<tr>
<td>Liste des domaines bloqués</td>
<td>Bloquer des domaines de courriel spécifiques.</td>
</tr>
</tbody>
</table>

### Objet de sortie

<table class="table">
<thead>
<tr>
<th><b>Propriété</b></th>
<th><b>Type</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>`valid`</td>
<td>Booléen</td>
<td>Renvoie `true` ou `false` selon que le courriel respecte ou non les règles de vérification.</td>
</tr>
<tr>
<td>`cause`</td>
<td>Chaîne</td>
<td>
Si la propriété « valid » est `false`, une référence est renvoyée à la première règle que le courriel n’a pas respectée :
            <ul> 
            <li> `MISSING_MX_RECORD:` Le domaine de courriel n’a pas d’enregistrements MX.</li>
<li> `FREE_EMAIL:` Le domaine de courriel est un fournisseur d’accès gratuit.</li>
<li> `DISPOSABLE_EMAIL:` Le courriel est une adresse temporaire.</li>
<li> `BLOCKLISTED:` Le domaine de courriel est inclus dans le paramètre Liste des domaines bloqués.</li>
<li> `NOT_ALLOWED:` Le domaine de courriel n’est pas inclus dans le paramètre Liste des domaines autorisés.</li>
</ul></td>
</tr>
</tbody>
</table>

### Exemple d’objet de sortie

```javascript lines
{
  "valid": false,
  "cause": "FREE_EMAIL"
}
```

```javascript lines
{
  "valid": true,
  "cause": null
}
```
