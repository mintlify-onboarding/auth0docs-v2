Cette liste d’actions de jetons Web JSON vous permet de générer, de vérifier et de décoder les <Tooltip href="/docs/fr-ca/glossary?term=json-web-token" tip="Jeton Web JSON (JWT)
Format standard de jeton d’ID (et souvent de jeton d’accès) utilisé pour représenter en toute sécurité des demandes entre deux parties." cta="Voir le glossaire">JWT</Tooltip> dans vos flux.

## Configurer la connexion à votre coffre-fort

Pour configurer une connexion au coffre-fort pour vos actions de jeton Web JSON utilisant **algorithme HS256**, vous aurez besoin d’un **Secret :**

<Frame>![Dashboard > Actions > Forms > Flows > actions > JSON web token](/docs/images/fr-ca/cdy7uua7fh8z/1WnUlIe2QZf39G45PHcQGp/40d768da226a0cefa09c62c5cab78577/jwt-vault-connection.png)</Frame>

## Signer un jeton Web JSON

Génère un jeton Web JSON.

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/3XQqUMKrmrc8fUkCsDhI2T/43f7f8d38673d8fa4f9dba7242e044d9/sign-json-web-token.png)</Frame>

### Paramètres de saisie

<table class="table">
<thead>
<tr>
<th><b>Paramètre</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>Charge utile</td>
<td>
              Données à encoder. Nous vous recommandons de les formater selon les normes OpenID.	
          </td>
</tr>
<tr>
<td>Objet</td>
<td>
             Identifie l’objet du JWT.
          </td>
</tr>
<tr>
<td>Émetteur</td>
<td>
              Identifie le mandant qui a émis le JWT.	
          </td>
</tr>
<tr>
<td>Audience</td>
<td>
             Identifie les destinataires auxquels le JWT est destiné. Par exemple :  Admin.your_domain.com
          </td>
</tr>
<tr>
<td>Expire dans</td>
<td>
             Identifie le délai d’expiration après lequel le JWT ne doit plus être accepté aux fins de traitement. 
          </td>
</tr>
</tbody>
</table>

### Objet de sortie

<table class="table">
<thead>
<tr>
<th><b>Propriété</b></th>
<th><b>Type</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>`token`</td>
<td>Chaîne</td>
<td>Une chaîne de jeton web JSON.</td>
</tr>
</tbody>
</table>

### Exemple d’objet de sortie

```json lines
{
  "token": "eyJhbGciOiJIUzI1N..."
}
```

## Décoder un jeton Web JSON

Décode un jeton Web JSON fourni.

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/7I25WVtppllW6qhC5sALvH/70143bbe0d96920e1d9dfabbb1d6aeff/decode-json-web-token.png)</Frame>

### Paramètres de saisie

<table class="table">
<thead>
<tr>
<th><b>Paramètre</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>Jeton (obligatoire)</td>
<td>
              Chaîne du jeton web JSON qui sera décodé.	
          </td>
</tr>
</tbody>
</table>

### Objet de sortie

<table class="table">
<thead>
<tr>
<th><b>Propriété</b></th>
<th><b>Type</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>`payload`</td>
<td>objet</td>
<td>Le contenu du jeton web JSON décodé et valide</td>
</tr>
</tbody>
</table>

### Exemple d’objet de sortie

```json lines
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "1234567890",
    "name": "John Doe",
    "iat": 1516239022
  },
  "signature": "SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c"
}
```

## Vérifier un jeton Web JSON

Vérifie les données du jeton Web JSON pour déterminer si elles sont intactes ou si elles ont été modifiées, afin d’en garantir l’authenticité.

<Frame>![](/docs/images/fr-ca/cdy7uua7fh8z/15MDJcfZqtC46h6mRPOGcz/55e0785db066a54f836d30199fa5f295/verify-json-web-token.png)</Frame>

### Paramètres de saisie

<table class="table">
<thead>
<tr>
<th><b>Paramètre</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>Jeton (obligatoire)</td>
<td>
              Chaîne de jeton Web JSON qui sera vérifiée.	
          </td>
</tr>
<tr>
<td>Émetteur</td>
<td>
              L’émetteur du JWT qui sera vérifié.	
          </td>
</tr>
<tr>
<td>Audience</td>
<td>
             L’audience destinataire du JWT est censé être vérifié.
          </td>
</tr>
</tbody>
</table>

### Objet de sortie

<table class="table">
<thead>
<tr>
<th><b>Propriété</b></th>
<th><b>Type</b></th>
<th><b>Description</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>`valid`</td>
<td>Booléen</td>
<td>Renvoie `true` ou `false` selon que le JWT possède ou non une signature valide.</td>
</tr>
<tr>
<td>`cause`</td>
<td>Chaîne</td>
<td>Si la propriété `valid` est `false` un message s’affiche.</td>
</tr>
<tr>
<td>`payload`</td>
<td>Objet</td>
<td>Le contenu du jeton Web JSON décodé et valide.</td>
</tr>
</tbody>
</table>

### Exemple d’objet de sortie

```json lines
{
  "valid": true,
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "1234567890",
    "name": "Jane Doe",
    "iat": 1516239022
  },
  "signature": "SflKxwRJSMe..."
}
```

```json lines
{
  "valid": false,
  "cause": "INVALID_SIGNATURE"
}
```
