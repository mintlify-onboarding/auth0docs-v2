La Connexion universelle consiste en un ensemble de pages qui facilitent plusieurs actions liées au compte, telles que la connexion, l’enrôlement des facteurs d’authentification multifacteur (<Tooltip href="/docs/fr-ca/glossary?term=multifactor-authentication" tip="Authentification multifacteur (MFA)
Processus d’authentification de l’utilisateur qui utilise un facteur en plus du nom d’utilisateur et du mot de passe, tel qu’un code par SMS." cta="Voir le glossaire">MFA</Tooltip>) ou la réinitialisation des mots de passe. Auth0 fournit le texte de ces pages en plusieurs langues.

Dans certains cas, vous pouvez modifier le texte de ces pages pour qu’il corresponde mieux au ton de votre application ou à vos besoins spécifiques.

Vous pouvez personnaliser le texte affiché sur les pages [Connexion universelle](/docs/fr-ca/authenticate/login/auth0-universal-login/universal-login-vs-classic-login/universal-experience) dans toutes les langues prises en charge par <Tooltip href="/docs/fr-ca/glossary?term=auth0-dashboard" tip="Auth0 Dashboard
Principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire">Auth0 Dashboard</Tooltip> ou <Tooltip href="/docs/fr-ca/glossary?term=management-api" tip="Management API
Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire">Management API</Tooltip>. Les définitions, valeurs et variables suivantes s’appliquent aux deux méthodes.

## Configuration des invites et des écrans

Vous pouvez personnaliser n’importe lequel des [prompts and screens (invites et écrans)](#prompt-values) disponibles via Auth0 Dashboard ou [Management API](/docs/fr-ca/api/management/v2).

<Tabs><Tab title="Utiliser le tableau de bord">

Vous pouvez utiliser le Tableau de bord pour personnaliser les invites pour le texte associé à la connexion universelle.

1. Dans le tableau de bord, accédez à [**Branding > Universal Login > Advanced Options (Marque > Connexion universelle > Options avancées)**](https://manage.auth0.com/#/custom_text) puis sélectionnez **Custom Text (Texte personnalisé)**.
2. Sélectionnez **l’invite** et **l’écran** que vous souhaitez modifier, puis effectuez les modifications souhaitées. Un aperçu en direct est disponible sur la droite.
3. Lorsque vous êtes prêt(e) à ce que vos modifications s’affichent dans votre environnement de production, sélectionnez **Save Changes (Enregistrer les modifications).**

</Tab><Tab title="Utiliser Management API">

Vous pouvez utiliser Management API pour personnaliser les invites pour le texte associé à la connexion universelle.

1. Obtenez un [jeton d’accès à Management API](/docs/fr-ca/secure/tokens/access-tokens/management-api-access-tokens/get-management-api-access-tokens-for-production) avec les permissions `read:prompts` et `update:prompts`.
2. Appelez le point de terminaison [Obtenir un texte personnalisé pour une invite](https://auth0.com/docs/api/management/v2#!/Prompts/get_custom_text_by_language) pour Management API pour obtenir les données de configuration existantes pour l’invite et la langue spécifiées.
3. Modifiez les données de configuration au besoin à l’aide des [paramètres pris en charge](#supported-parameters) et [des valeurs d’invite](#prompt-values) comme références.
4. Appelez le point de terminaison [Définir du texte personnalisé pour une invite donnée](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) de Management API avec les données de configuration mises à jour.

   <Warning>

   Le point de terminaison [Set custom text for a specific prompt (Définir un texte personnalisé pour une invite spécifique)](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) de Management API remplace toutes les données de configuration existantes pour l’invite spécifiée par les données de configuration fournies dans le paramètre `body (corps)`.

   Si vous voulez configurer de multiples valeurs `screen (écran)` pour une invite spécifiée, vous devez toutes les inclure dans un seul appel à Management API.

   </Warning>

#### Exemple

Vous pourriez configurer le composant de texte `description` de l’invite `login` (connexion) sur l’écran de `connexion` pour `se connecter au site Web d’ACME` en appelant le point de terminaison [Set custom text for a specific prompt](https://auth0.com/docs/api/management/v2#!/Prompts/put_custom_text_by_language) (Définir un texte personnalisé pour une invite donnée) de Management API de la façon suivante :

<AuthCodeGroup>
```bash cURL
curl --request PUT \
  --url 'https://{yourDomain}/api/v2/prompts/login/custom-text/en' \
  --header 'authorization: Bearer {yourMgmtApiAccessToken}' \
  --header 'content-type: application/json' \
  --data '{ "login": { "description": "Login to ACME'''s Website" } }'
```
```csharp C#
var client = new RestClient("https://{yourDomain}/api/v2/prompts/login/custom-text/en");
var request = new RestRequest(Method.PUT);
request.AddHeader("content-type", "application/json");
request.AddHeader("authorization", "Bearer {yourMgmtApiAccessToken}");
request.AddParameter("application/json", "{ "login": { "description": "Login to ACME's Website" } }", ParameterType.RequestBody);
IRestResponse response = client.Execute(request);
```
```go Go
package main

import (
	"fmt"
	"strings"
	"net/http"
	"io/ioutil"
)

func main() {

	url := "https://{yourDomain}/api/v2/prompts/login/custom-text/en"

	payload := strings.NewReader("{ "login": { "description": "Login to ACME's Website" } }")

	req, _ := http.NewRequest("PUT", url, payload)

	req.Header.Add("content-type", "application/json")
	req.Header.Add("authorization", "Bearer {yourMgmtApiAccessToken}")

	res, _ := http.DefaultClient.Do(req)

	defer res.Body.Close()
	body, _ := ioutil.ReadAll(res.Body)

	fmt.Println(res)
	fmt.Println(string(body))

}
```

### Valeurs des invites

Le terme `invite` fait référence à une étape spécifique du flux de connexion. Chaque `invite` possède au moins un `écran` et chaque `écran` possède un ensemble de composants textuels que vous pouvez personnaliser.

Les valeurs d’`invite` disponibles sont :

## Variables

Certains écrans comportent des variables dans le texte qui sont remplacées au moment de l’exécution en fonction des informations contextuelles. Comme différentes variables sont disponibles pour chaque écran, il n’est pas garanti qu’une variable donnée fonctionne ailleurs que dans son écran d’origine.

<table class="table"><thead>
<tr>
<th><strong>Variable</strong></th>
<th><strong>Description</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td>`${clientName}`</td>
<td>Nom de l’application Auth0</td>
</tr>
<tr>
<td>`${connectionName}`</td>
<td>Nom de la connexion (par ex., Google)</td>
</tr>
<tr>
<td>`${tenantName}`</td>
<td>Nom du locataire Auth0</td>
</tr>
<tr>
<td>`${organizationName}`</td>

</tr>

<td>`${companyName}`</td>
<td>Nom du locataire, ou nom de l’Organization si l’utilisateur s’authentifie dans le contexte d’une Organization</td>
</tr>
<tr>
<td>`${userName}`</td>
<td>Nom de l’utilisateur connecté</td>
</tr>
