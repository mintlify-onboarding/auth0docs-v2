Dans ce scénario, chaque organisation exploitant la fonctionnalité [Auth0 Organizations](/docs/fr-ca/manage-users/organizations/organizations-overview) dispose exactement d’une connexion Auth0. Cette fonctionnalité permet à chaque application de sélectionner l’organization correspondante configurée sur le locataire Auth0. Cette organization authentifie directement les informations d’identification de l’utilisateur ou redirige la demande d’authentification vers l’<Tooltip href="/docs/fr-ca/glossary?term=idp" tip="Fournisseur d’identité (IdP)
Service de stockage et de gestion des identités numériques." cta="Voir le glossaire">IdP</Tooltip> approprié pour traitement.

Dans ce scénario, nous continuerons à utiliser l’exemple [Travel0 Corporate Booking](/docs/fr-ca/get-started/architecture-scenarios/multiple-organization-architecture). Dans cet exemple, l’application de réservation d’entreprise correspond à votre application et Travel0 correspond à votre entreprise.

<Frame>![Architecture Scenarios - Multitenancy - Diagram - Isolated users, Shared Apps, Orgs A&B](/docs/images/fr-ca/cdy7uua7fh8z/1YBaMaMozamiTv8hXmNB2D/21e9270a59021a12209bdcb4345147cb/isolated-users_shared-apps_org-ab.png)</Frame>

<Warning>

La fonctionnalité Auth0 Organizations (Organisations Auth0) prend actuellement en charge les applications qui utilisent les protocoles [SAML](/docs/fr-ca/authenticate/protocols/saml/saml-configuration), [Open ID Connect](/docs/fr-ca/authenticate/protocols/openid-connect-protocol) et/ou le cadre d’applications [OAuth 2](/docs/fr-ca/authenticate/protocols/oauth). Les applications necessitant [WS-Fed](/docs/fr-ca/authenticate/protocols/ws-fed-protocol) ne sont pas prises en charge.

</Warning>

Peu importe la méthode utilisée pour traiter les informations d’identification, qu’il s’agisse d’une connexion à une base de données (comme avec Hoekstra & Associates), d’une validation via une connexion d’entreprise (comme avec MetaHexa Bank) ou d’une connexion sociale, votre application fonctionne de manière identique dans tous les cas. Tirer parti de la fonctionnalité [Organizations](/docs/fr-ca/manage-users/organizations/organizations-overview) vous permet de configurer et de contrôler l’expérience.
