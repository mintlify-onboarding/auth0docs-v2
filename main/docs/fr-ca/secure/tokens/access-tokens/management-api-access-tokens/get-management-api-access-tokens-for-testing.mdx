<Warning>

Cette méthode d’obtention des jetons d’accès est utilisée **à des fins de test uniquement**. Ne vous procurez pas manuellement des jetons à durée de vie longue et ne les utilisez pas dans vos applications, car cela annule les avantages relatifs à la sécurité que les jetons offrent.

</Warning>

Lorsque vous terminez la configuration dans <Tooltip href="/docs/fr-ca/glossary?term=auth0-dashboard" tip="Auth0 Dashboard
Principal produit d’Auth0 pour configurer vos services." cta="Voir le glossaire">Auth0 Dashboard</Tooltip>, vous obtenez un jeton de test pour <Tooltip href="/docs/fr-ca/glossary?term=management-api" tip="Management API
Un produit permettant aux clients d’effectuer des tâches administratives." cta="Voir le glossaire">Management API</Tooltip>. Vous ne devez le faire qu’une fois, sauf si vous créez un nouveau locataire. Nous vous recommandons de créer un jeton de test exclusivement pour autoriser l’accès à Management API au lieu de réutiliser un autre jeton que vous pourriez avoir.

## Prérequis

* [Créer des applications machine-machine pour les tests](/docs/fr-ca/get-started/apis/create-m2m-app-test)

## Obtenir des jetons d’accès manuellement

1. Dans votre Auth0 Dashboard, naviguez jusqu’à **Applications &gt API >** [Auth0 Management API](https://manage.auth0.com/#/apis/management/explorer). Sélectionnez l’onglet **Explorateur d’API**  et localisez un jeton généré automatiquement dans la section **Jeton**.
2. Sélectionnez l’icône **Copier** à droite du jeton. Vous pouvez maintenant effectuer des appels autorisés à la [Management API](/docs/fr-ca/api/management/v2) à l’aide de ce jeton.

   <Frame>![APIs - Management API access tokens - Testing - Copy token](/docs/images/fr-ca/cdy7uua7fh8z/3Y4I9YJR77j73FCYDiegnC/108365ff5fd10764cd633a4be905d1e1/Mgmt_API_Explorer.png)</Frame>

3. , mettez à jour **l’expiration du jeton (Secondes)** et cliquez sur **Update &amp (Mettre à jour & Regenerate Token (Régénérer le jeton)**.

<Warning>

Ces jetons **ne peuvent pas être révoqués**. De longues durées d’expiration ne sont donc pas conseillées. Nous vous conseillons plutôt d’utiliser des délais d’expiration courts et d’en émettre un nouveau à chaque fois que vous en avez besoin. Vous pouvez vérifier le temps d’expiration en utilisant [jwt.io](https://jwt.io/) pour décoder le jeton.

</Warning>

## Utiliser des jetons d’accès pour les tests

Vous pouvez utiliser le jeton d’accès que vous venez de créer à des fins de test pour appeler manuellement un point de terminaison via l’[Explorateur de Management API](/docs/fr-ca/api/management/v2).

1. Accédez à l’[Explorateur de Management API](/docs/fr-ca/api/management/v2#!) et sélectionnez le bouton **Définir le jeton API**.
2. Remplissez le champ **Token API**. Des informations supplémentaires s’affichent alors dans la fenêtre contextuelle, telles que les permissions accordées à l’application.
3. Sélectionnez **Définir le jeton API** pour enregistrer votre jeton et fermer la fenêtre contextuelle. Remarquez que le bouton Définir le jeton API s’affiche désormais sous la forme **Autorisé**.
4. Pour commencer les tests, naviguez jusqu’à un point de terminaison que vous souhaitez appeler et saisissez tous les paramètres requis ou facultatifs dans les champs de droite. Ensuite, sélectionnez **Tester le point de terminaison**.
