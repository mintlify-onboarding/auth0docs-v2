By default, seamless <Tooltip tip="Single Sign-On (SSO): Service that, after a user logs into one applicaton, automatically logs that user in to other applications." cta="View Glossary" href="/glossary?term=single+sign-on">single sign-on</Tooltip> (SSO) is enabled for all new Auth0 tenants; however, **legacy tenants** may [choose whether to enable this feature at the tenant level](/get-started/tenant-settings/enable-sso-for-legacy-tenants). If you have not enabled tenant-level SSO, you may enable it per application.

You can enable an SSO for your application using the <Tooltip tip="Auth0 Dashboard: Auth0's main product to configure your services." cta="View Glossary" href="/glossary?term=Auth0+Dashboard">Auth0 Dashboard</Tooltip>.

## Prerequisite

Before enabling SSO for an application, you must first create and configure a connection for each <Tooltip tip="Identity Provider (IdP): Service that stores and manages digital identities." cta="View Glossary" href="/glossary?term=identity+provider">identity provider</Tooltip> you want to use. For social identity providers, make sure the connection is not using developer keys if you use the [Universal Login experience](/authenticate/login/auth0-universal-login).

## Enable SSO

1. Go to [Auth0 Dashboard > Applications > SSO Integrations](https://manage.auth0.com/#/externalapps), and select the name of the integration to view.
2. Scroll to the bottom of the **Settings** page, locate **Use Auth0 instead of the IdP to do Single Sign-on**, enable the toggle, and select **Save Changes**.
