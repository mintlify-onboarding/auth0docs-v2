「[カスタムエラーページ](/docs/ja-jp/customize/login-pages/custom-error-pages)」をお読みください。

汎用エラーページのデフォルトのページ名を探すには、「[バージョン管理を使用してユニバーサルログインページを管理する](/docs/ja-jp/customize/login-pages/classic-login/version-control)」をご覧ください。

## Display（表示）

[［Tenant Settings（テナント設定）］](https://manage.auth0.com/#/tenant/)で関連するフィールドを構成済みの場合、汎用エラーページに以下の情報が表示されます。

<table class="table"><thead>
<tr>
<th>フィールド</th>
<th>説明</th>
</tr>
</thead>
<tbody>
<tr>
<td>Friendly Name（フレンドリー名）</td>
<td>会社の名前</td>
</tr>
<tr>
<td>Logo URL（ロゴURL）</td>
<td>会社ロゴのURL</td>
</tr>
<tr>
<td>Support Email（サポートメール）</td>
<td>会社のサポートチームのメールアドレス</td>
</tr>
<tr>
<td>Support URL（サポートURL）</td>
<td>会社のサポートページのURL</td>
</tr>
</tbody>
</table>

## パラメーター

認可サーバーは、パラメーターをクエリ文字列としてURLに追加して返します。

<table class="table"><thead>
<tr>
<th><b>パラメーター</b></th>
<th><b>説明</b></th>
</tr>
</thead>
<tbody>
<tr>
<td>`client_id`</td>
<td>Auth0アプリケーションの識別子。</td>
</tr>
<tr>
<td>`connection`</td>
<td>エラー時に使用された接続。</td>
</tr>
<tr>
<td>`lang`</td>
<td>エラー時に使用された言語セット。</td>
</tr>
<tr>
<td>`error`</td>
<td>エラーコード。</td>
</tr>
<tr>
<td>`error_description`</td>
<td>エラーの説明。</td>
</tr>
<tr>
<td>`tracking`</td>
<td>内部ログでエラーを見つけるためにAuth0で使用される識別子。</td>
</tr>
</tbody>
</table>

このパラメーターはエラーの種類によって異なり、要求固有のものになります。たとえば、要求に`client_id`が含まれておらずエラーが発生した場合、エラーページでクライアントIDは返されません。
