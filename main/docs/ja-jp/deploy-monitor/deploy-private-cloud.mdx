プライベートクラウドパッケージは、以下の場合に使用できるマネージドサービスです。

* 組織でマルチテナントのパブリッククラウドサービスを使用できない
* パブリッククラウドのオプションでは実現できない地理的なデプロイメント要件がある
* 保証されたRPS（1秒間に処理可能な要求数）のレベル、またはパブリッククラウドで利用できる以上の容量が必要である

Auth0プライベートクラウドは以下の方法で導入できます。

<table class="table"><thead>
<tr>
<th><strong>オプション</strong></th>
<th><strong>説明</strong></th>
</tr>
</thead>
<tbody>
<tr>
<td><a href="/docs/ja-jp/deploy/private-cloud"><strong>AWSのプライベートクラウド</strong></a></td>
<td>AWSで稼働する専用のマネージドクラウドサービス。隔離、高パフォーマンス、開発インスタンス、Geo-HAアドオンなど多数の機能を提供</td>
</tr>
<tr>
<td><a href="/docs/ja-jp/deploy-monitor/deploy-private-cloud/private-cloud-on-azure"><strong>Azureのプライベートクラウド</strong></a></td>
<td>Azureで稼働する専用クラウドサービス。隔離、高パフォーマンス、開発インスタンス、アドオンなど多数の機能を提供</td>
</tr>
</tbody>
</table>

また、Auth0プライベートクラウドの導入を補完する[アドオン機能](/docs/ja-jp/deploy-monitor/deploy-private-cloud/private-cloud-add-on-features)も提供しています。

特定のサポート要件がある場合や、当社が提供するプロフェッショナルサービスに関する詳細を希望される場合は、[Auth0営業担当](https://auth0.com/contact-us)までお問い合わせください。
