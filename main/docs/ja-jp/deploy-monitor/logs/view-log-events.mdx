<Tooltip data-tooltip-id="react-containers-DefinitionTooltip-0" href="/docs/ja-jp/glossary?term=auth0-dashboard" tip="Auth0 Dashboard: サービスを構成するためのAuth0の主製品。" cta="用語集の表示">Auth0 Dashboard</Tooltip>を使用して、テナントで発生するすべてのイベント（ユーザー認証やアプリケーション、接続、ルールの追加・更新などの管理アクションを含む）を表示します。管理アクションは**API操作** イベントとしてログに表示されます。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">

ログイベントは、デフォルトでは必ずしも発生した順序で表示されません。テナントログをストリーミングしていて、発生順に並べ替えたい場合は、ストリーム処理でログの並べ替えを行うことをお勧めします。

</Callout>

1. [［Dashboard］>［Monitoring（モニタリング）］>［Logs（ログ）］](https://manage.auth0.com/#/logs)に移動します。

   <Frame>![Dashboard - モニタリング - ログ](/docs/images/ja-jp/cdy7uua7fh8z/2l9LfKEjrprVmHfWt80VX0/44fb4d1dad15f13ceb21e64d093ebf5c/2025-01-28_15-02-44.png)</Frame>

2. リストには最新のイベントが表示されます。検索を行うには、カーソルを検索フィールドに置き、クエリを入力して、**Enter** を押します。たとえば、"Pass"を含む接続を使ったすべてのログを検索するには、`connection:*pass*`と入力します。検索クエリ構文と例の詳細については、「[ログ検索クエリ構文](/docs/ja-jp/deploy-monitor/logs/log-search-query-syntax)」をお読みください。
3. 検索はイベントタイプごとにフィルター処理することができます。ドロップダウンリストからフィルターを選択し、**［Apply（適用）］** をクリックします。たとえば、失敗したユーザーログインイベントにフィルターを適用するには、**［Failed Login（失敗したログイン）］** を選択します。ログイベントフィルターと例の詳細については、「[ログイベントフィルター](/docs/ja-jp/deploy-monitor/logs/log-event-filters)」をお読みください。

   <Frame>![Dashboard Monitoring Logs Filters](/docs/images/ja-jp/cdy7uua7fh8z/3bfjw3WURyQRiNhj6f8VlJ/5c71550ff0a1d51f81b0c65a4d2cc22a/dashboard-monitoring-logs-filters.png)</Frame>

4. 検索で日付範囲を選択することができます。日付フィールドの横のドロップダウンをクリックし、特定の日付または範囲を選択し、**［Apply（適用）］** をクリックします。
