以下のチェックでは、生産導入を成功させるための推奨事項（オプション）をすべて完了したかどうかを確認します。

チェックの出力の詳細については、[［Run Production Checks（生産チェックの実行）］](/docs/ja-jp/deploy-monitor/pre-deployment-checks/how-to-run-production-checks)を参照してください。

<table class="table"><thead>
<tr>
<th>確認</th>
<th>説明</th>
</tr>
</thead>
<tbody>
<tr>
<td>ログストリーミングの構成</td>
<td>テナントログエントリを外部ストレージ場所またはプロバイダーにストリームしていることを確認する。<a href="https://auth0.com/docs/ja-jp/customize/log-streams">ログストリーミングサービスを構成</a>して、ログを30日間以上保持し、分析およびアラートを向上することをお勧めします。</td>
</tr>
<tr>
<td><a href="/docs/ja-jp/custom-domains">カスタムドメイン</a>の構成</td>
<td><a href="https://manage.auth0.com/#/tenant/custom_domains">ユニバーサルログインでカスタムドメイン</a>を使用して、エンドユーザーによりシームレスで安全なエクスペリエンスを提供します。</td>
</tr>
<tr>
<td><a href="/docs/ja-jp/universal-login/custom-error-pages">カスタムエラーページ</a>の構成</td>
<td><a href="https://manage.auth0.com/#/account">アプリケーション固有の詳細とコーポレートブランディングでカスタムエラーページ</a>を構成します。</td>
</tr>
<tr>
<td>ユニバーサルログインの有効化</td>
<td>最新バージョンの<a href="https://auth0.com/docs/ja-jp/authenticate/login/auth0-universal-login/new-universal-login-vs-classic-universal-login">ユニバーサルログイン</a>を使用していることを確認します。向上したアクセシビリティやパフォーマンス、ノーコードカスタマイゼーション、新機能の恩恵を受けるために、クラシックのログインページからユニバーサルログインに更新することをお勧めします。</td>
</tr>
<tr>
<td><a href="/docs/ja-jp/email/custom">メールテンプレート</a>の構成</td>
<td>アプリケーション固有の詳細とコーポレートブランディングで<a href="https://manage.auth0.com/#/emails">カスタムメールテンプレートを構成</a>します。</td>
</tr>
<tr>
<td><a href="/docs/ja-jp/mfa">多要素認証</a>の構成</td>
<td>ユーザーの認証のセキュリティプラクティスとして、多要素認証を検討することをお勧めします。</td>
</tr>
<tr>
<td><a href="/docs/ja-jp/tutorials/manage-dashboard-admins">テナント管理者のためのMFA</a>を有効にする</td>
<td><a href="https://manage.auth0.com/#/account/admins">テナント管理者のために多要素認証</a>を有効にします。</td>
</tr>
<tr>
<td><a href="/docs/ja-jp/logout#set-the-allowed-logout-urls-at-the-account-level">リダイレクトログアウト URL</a>の定義</td>
<td>アプリケーションの<a href="https://manage.auth0.com/#/account/advanced">許可されているリダイレクトログアウトURL</a>を確認します。</td>
</tr>
<tr>
<td>アプリケーションはJSON Web Token（JWT）署名アルゴリズムのためにHS256ではなくRS256を使う必要がある</td>
<td>各アプリケーションに対してJSON Web Token（JWT）<a href="/docs/ja-jp/apis#signing-algorithms">署名アルゴリズム</a>をHS256ではなくRS256に設定します。</td>
</tr>
<tr>
<td><a href="https://auth0.com/docs/ja-jp/get-started/applications/application-settings#application-uris">アプリケーションログインURI</a>の設定</td>
<td>すべての対話型アプリケーションに対して<a href="https://auth0.com/docs/ja-jp/get-started/applications/application-settings#application-uris">アプリケーションログインURI</a>を指定します。</td>
</tr>
<tr>
<td><a href="https://auth0.com/docs/ja-jp/get-started/applications/application-settings#application-uris">テナント許可ログアウトURL</a>の設定</td>
<td>1つ以上のパブリックアクセス可能なログアウトルートを作成し、そこを指すように<a href="https://auth0.com/docs/ja-jp/get-started/applications/application-settings#application-uris">テナント設定を更新します</a>。</td>
</tr>
<tr>
<td><a href="https://auth0.com/docs/ja-jp/get-started/tenant-settings#login-and-logout">テナントログインURI</a>の設定</td>
<td>デフォルトのログインルートを公開し、そこを指すように<a href="https://auth0.com/docs/ja-jp/get-started/tenant-settings#login-and-logout">テナント設定を更新します</a>。</td>
</tr>
</tbody>
</table>
